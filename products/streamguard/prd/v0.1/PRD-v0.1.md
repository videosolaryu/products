## 1. 产品定位

### 1.1 一句话定义
StreamGuard 是一款面向互联网直播信号的私有化部署监测与可视化平台，支持多协议信号接入、实时质量指标分析、多路合屏展示，为运维团队提供 7×24 小时的信号健康监控能力。

### 1.2 目标用户
- **核心用户**: 直播平台运维团队、CDN 服务商、企业直播部门
- **典型场景**: 
  - 多路直播源统一监控（推流/拉流）
  - 直播质量实时告警与故障定位
  - 多画面合屏展示供值班人员查看

### 1.3 核心价值主张
- **私有化部署**: 纯软件部署，数据不出内网
- **统一接入**: 支持 RTMP 推流接入
- **实时监测**: 码率、帧率等关键指标秒级采集
- **智能告警**: 告警生命周期管理，故障期间自动收敛
- **可视化合屏**: 灵活布局的多画面监控墙

## 3. 功能规格

### 3.1 核心功能

#### 3.1.1 信源管理

**信源配置**
| 功能 | 描述 | 验收标准 |
|------|------|----------|
| RTMP 推流接入 | 接收 RTMP 推流信号 | 支持 OBS/FFmpeg 推流，稳定接收 |
| 信源配置管理 | 添加/编辑/删除信源 | CRUD 完整，配置持久化 |
| 自动重连策略 | 配置重连间隔和退避策略 | 支持固定间隔/指数退避 |

**注：HLS/SRT/RTSP协议支持移至v0.2，详见Backlog**

**监控任务管理**
| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 监控任务列表 | 独立列表展示当前监控中的信源 | 与信源配置分离，仅显示已启动监控的任务 |
| 启动/停止监控 | 对信源开启或关闭实时监控 | 操作即时生效，状态同步更新 |
| 任务状态显示 | 显示监控任务的运行状态 | 运行中/已停止/异常，实时刷新 |
| 批量操作 | 支持批量启动/停止监控 | 支持多选，操作结果反馈 |

**信源 vs 监控任务区别**
| 维度 | 信源配置 | 监控任务 |
|------|----------|----------|
| 作用 | 定义信源参数（地址、协议等） | 控制是否对该信源进行实时监测 |
| 数据 | 静态配置信息 | 动态运行状态 |
| 生命周期 | 长期保存 | 可启停，异常时可能中断 |
| 数量关系 | 1个信源可对应0或1个监控任务 | 1个监控任务必须绑定1个信源 |

#### 3.1.3 告警系统

**告警类型（v0.1）**
| 告警类型 | 触发条件 | 级别 |
|----------|----------|------|
| 信号离线 | 信号断开或无法连接 | 异常 |
| 码率异常 | 码率过高/过低/剧烈波动 | 异常 |
| 帧率异常 | 帧率低于阈值 | 异常 |
| **黑帧** | 画面全黑持续>2秒 | 异常 |
| **静帧** | 画面静止持续>5秒 | 异常 |
| **音频静音** | 电平<-60dB持续5秒 | 警告 |

**注：音频响度超标、音频过载、TS同步错误、CC错误告警移至v0.2**

**告警生命周期**
```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  正常   │───▶│  异常   │───▶│  恢复   │
└─────────┘    └────┬────┘    └─────────┘
                    │
                    ▼
              ┌─────────────┐
              │ 生成告警记录 │
              │ 状态:未恢复  │
              └─────────────┘
                    │
                    ▼
              ┌─────────────┐
              │ 故障持续期间 │
              │ 不重复生成   │
              └─────────────┘
                    │
                    ▼
              ┌─────────────┐
              │ 故障恢复     │
              │ 更新原记录   │
              │ 状态:已恢复  │
              │ 记录恢复时间 │
              └─────────────┘
```

**告警去重逻辑**
- 同一信源同一类型 = 同一告警
- 离线→恢复→再次离线 = 两条独立告警
- 离线期间其他异常 = 只报离线（根因优先）

**告警通知**
- Webhook HTTP 回调
- 支持自定义 Payload 格式
- **声音告警**: 蜂鸣器/自定义音频文件

| 字段 | 类型 | 说明 |

6. **系统设置**: 告警规则、数据保留期、系统配置

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| FFmpeg 协议兼容性 | 高 | 充分测试主流推流工具，建立兼容性矩阵 |
| 响度算法实现复杂度 | 中 | 采用成熟开源库(ebur128)，充分测试验证 |
| 黑帧/静帧检测性能 | 中 | GPU加速可选，CPU优化算法 |
| 高并发 CPU 瓶颈 | 高 | 早期压力测试，预留 GPU 加速扩展 |
| 浏览器解码性能 | 中 | 控制单屏路数，预留 WebRTC 升级 |
| 竞品价格竞争 | 中 | 开源核心 + 企业授权差异化定价 |

## 10. 附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| RTMP | Real-Time Messaging Protocol，实时消息传输协议 |
| HLS | HTTP Live Streaming，苹果提出的流媒体协议 |
| SRT | Secure Reliable Transport，安全可靠传输协议 |
| RTSP | Real Time Streaming Protocol，实时流传输协议 |
| **R-128** | **EBU响度标准，目标-23 LUFS** |
| **LUFS** | **响度单位，相对于满刻度分贝** |
| **LRA** | **响度范围，Loudness Range** |
| **True Peak** | **真峰值，高于数字峰值的实际峰值** |
| **TR-101-290** | **DVB标准，TS流错误检测规范** |
| CC | Continuity Counter，连续性计数器 |
| PCR | Program Clock Reference，节目时钟参考 |
| GOP | Group of Pictures，视频编码中的图像组 |
| CBR/VBR | 恒定/可变码率 |
| On-Premise | 私有化部署，软件部署在客户自有服务器 |
